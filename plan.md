# Chatterbox3000: Cold Email Filter & Approval System

## Overview

A self-hosted email filtering service that sits between incoming mail and your inbox. Incoming emails are intercepted by Cloudflare Email Workers, classified, and either auto-forwarded or held for manual review in a Chrome Extension sidebar.

---

## Architecture

```
Sender
  │
  ▼
MX → Cloudflare Email Routing
  │
  ▼
Email Worker
  ├── Parse email (from, subject, body preview, SPF/DKIM status)
  ├── Check sender against allowlist/blocklist (D1)
  ├── If allowed → forward immediately to destination
  ├── If blocked → drop silently or store as rejected
  └── If unknown → store in D1 as "pending", send push event via Durable Object
        │
        ▼
      D1 Database
        │
        ▲
REST API Worker ←── Chrome Extension (Side Panel)
  ├── GET  /api/emails?status=pending
  ├── POST /api/emails/:id/approve
  ├── POST /api/emails/:id/reject
  ├── GET  /api/senders
  ├── POST /api/senders/:address/allow
  ├── POST /api/senders/:address/block
  ├── GET  /api/settings
  ├── PUT  /api/settings
  └── WebSocket via Durable Object (real-time updates)
```

## Database Schema (Drizzle + D1)

### Drizzle config (`worker/drizzle.config.ts`)

```ts

export default defineConfig({
  schema: "./src/db/schema.ts",
  out: "./src/db/migrations",
  dialect: "sqlite",
  driver: "d1-http",
});
```

### Schema (`worker/src/db/schema.ts`)

```ts
import { sqliteTable, text, integer } from "drizzle-orm/sqlite-core";
import { sql } from "drizzle-orm";

export const emails = sqliteTable("emails", {
  id: text("id")
    .primaryKey()
    .$defaultFn(() => crypto.randomUUID()),
  messageId: text("message_id"),
  sender: text("sender").notNull(),
  fromHeader: text("from_header").notNull(),
  toHeader: text("to_header").notNull(),
  subject: text("subject").notNull().default(""),
  bodyPreview: text("body_preview").default(""),
  rawSize: integer("raw_size").default(0),
  spfPass: integer("spf_pass", { mode: "boolean" }).default(false),
  dkimPass: integer("dkim_pass", { mode: "boolean" }).default(false),
  status: text("status", {
    enum: ["pending", "approved", "rejected", "forwarded"],
  })
    .notNull()
    .default("pending"),
  classification: text("classification", {
    enum: ["cold_outreach", "spam", "newsletter", "legitimate", "unknown"],
  }),
  classificationReason: text("classification_reason"),
  rawStorageKey: text("raw_storage_key"),
  createdAt: text("created_at")
    .notNull()
    .default(sql`(datetime('now'))`),
  actionedAt: text("actioned_at"),
});

export const senders = sqliteTable("senders", {
  address: text("address").primaryKey(),
  displayName: text("display_name"),
  status: text("status", {
    enum: ["allowed", "blocked", "unknown"],
  })
    .notNull()
    .default("unknown"),
  emailCount: integer("email_count").default(0),
  firstSeen: text("first_seen")
    .notNull()
    .default(sql`(datetime('now'))`),
  lastSeen: text("last_seen")
    .notNull()
    .default(sql`(datetime('now'))`),
  notes: text("notes"),
});

export const settings = sqliteTable("settings", {
  key: text("key").primaryKey(),
  value: text("value").notNull(),
});
```

### Client factory (`worker/src/db/index.ts`)

```ts
import { drizzle } from "drizzle-orm/d1";
import * as schema from "./schema";

export function createDb(d1: D1Database) {
  return drizzle(d1, { schema });
}

export type Database = ReturnType<typeof createDb>;
```

### Example queries (`worker/src/services/storage.ts`)

```ts
import { eq, desc, and, sql } from "drizzle-orm";
import { emails, senders } from "../db/schema";
import type { Database } from "../db";

export async function getPendingEmails(db: Database) {
  return db.query.emails.findMany({
    where: eq(emails.status, "pending"),
    orderBy: desc(emails.createdAt),
  });
}

export async function approveEmail(db: Database, id: string) {
  return db
    .update(emails)
    .set({ status: "approved", actionedAt: sql`(datetime('now'))` })
    .where(eq(emails.id, id));
}

export async function upsertSender(
  db: Database,
  address: string,
  displayName: string | null,
) {
  return db
    .insert(senders)
    .values({ address, displayName })
    .onConflictDoUpdate({
      target: senders.address,
      set: {
        lastSeen: sql`(datetime('now'))`,
        emailCount: sql`${senders.emailCount} + 1`,
      },
    });
}

export async function allowSenderAndApproveEmails(
  db: Database,
  address: string,
) {
  await db
    .update(senders)
    .set({ status: "allowed" })
    .where(eq(senders.address, address));

  return db
    .update(emails)
    .set({ status: "approved", actionedAt: sql`(datetime('now'))` })
    .where(and(eq(emails.sender, address), eq(emails.status, "pending")));
}
```

## Component Detail

### 1. Email Worker (`email-handler.ts`)

Triggered by Cloudflare Email Routing for every incoming message.

```
Receive email event
  │
  ├── Extract: sender, from, to, subject, body preview
  ├── Extract: SPF/DKIM from event headers
  │
  ├── Lookup sender in `senders` table
  │   ├── If sender.status === 'allowed' AND auto_forward_allowed
  │   │   → Forward immediately via message.forward(destination)
  │   │   → Store email record with status='forwarded'
  │   │   → Update sender.last_seen, email_count++
  │   │
  │   ├── If sender.status === 'blocked' AND auto_reject_blocked
  │   │   → Store email record with status='rejected'
  │   │   → Update sender.last_seen, email_count++
  │   │   → Drop (don't forward)
  │   │
  │   └── If sender.status === 'unknown' OR no auto-action
  │       → Run classifier
  │       → Store email with status='pending'
  │       → Upsert sender as 'unknown'
  │       → Notify via Durable Object (WebSocket push)
  │
  └── Optionally store raw MIME in R2 for later replay
```

**Important:** `message.forward()` must be called within the email handler's lifetime. For queued emails that are approved later, you cannot use `message.forward()` — instead, you either:

- Store the raw MIME in R2 and re-send via an external SMTP relay (e.g., Resend, Mailchannels, or AWS SES) on approval
- Or use Cloudflare's `fetch()` to a mail-sending API

This is a critical design constraint — plan for an SMTP relay for deferred forwarding.

### 2. Classifier (`classifier.ts`)

Lightweight heuristic classifier that runs inline in the email worker. No LLM call needed for v1.

**Rules (applied in order):**

1. SPF fail AND DKIM fail → `spam` (high confidence)
2. Subject matches cold outreach patterns → `cold_outreach`
   - Patterns: "quick question", "following up", "partnership", "15 min call", "scale your", "grow your", "looking to connect", "[First name]?" as subject, etc.
3. Has `List-Unsubscribe` header → `newsletter`
4. Body contains calendar link + "book a time" / "schedule a call" → `cold_outreach`
5. Sender domain is freemail (gmail, outlook, yahoo) + subject is vague → `unknown`
6. Everything else → `legitimate`

**V2 enhancement:** Optional Workers AI call (e.g., `@cf/meta/llama-3-8b-instruct`) for borderline cases. Keep the prompt simple: "Classify this email as cold_outreach, spam, newsletter, or legitimate. Respond with one word."

### 3. API Router (`router.ts` — Hono)

All endpoints require `Authorization: Bearer <token>` header.

| Method | Path                        | Description                         |
| ------ | --------------------------- | ----------------------------------- |
| GET    | /api/emails                 | List emails, filter by `?status=`   |
| GET    | /api/emails/:id             | Get single email detail             |
| POST   | /api/emails/:id/approve     | Approve + trigger forward           |
| POST   | /api/emails/:id/reject      | Reject                              |
| POST   | /api/emails/bulk            | Bulk approve/reject `{ids, action}` |
| GET    | /api/senders                | List senders, filter by `?status=`  |
| POST   | /api/senders/:address/allow | Allowlist sender + approve pending  |
| POST   | /api/senders/:address/block | Blocklist sender + reject pending   |
| DELETE | /api/senders/:address       | Reset sender to unknown             |
| GET    | /api/settings               | Get settings                        |
| PUT    | /api/settings               | Update settings                     |
| GET    | /api/stats                  | Counts by status, last 24h/7d/30d   |
| GET    | /ws                         | WebSocket upgrade (Durable Object)  |

**Approve flow detail:**

1. Update email status to `approved`
2. If sender.status is still `unknown`, optionally prompt: "Also allow future emails from this sender?"
3. Retrieve raw MIME from R2
4. Send via SMTP relay to destination address
5. Update email status to `forwarded`

### 4. Chrome Extension — Side Panel

**Manifest V3 side panel** — opens as a sidebar within any tab.

#### Routes

**`/pending`** (default view)

- List of pending emails, newest first
- Each card shows: sender, subject, body preview, timestamp, classification badge
- Actions per email: Approve, Reject
- Actions per sender (from card): "Always allow this sender", "Always block this sender"
- Badge on extension icon shows pending count
- Real-time updates via WebSocket

**`/approved`**

- Log of approved/forwarded emails
- Searchable

**`/rejected`**

- Log of rejected emails
- "Undo" action (moves back to pending)

**`/senders`**

- Full sender list with status toggles (allowed / blocked / unknown)
- Search by address or display name
- Bulk actions
- Email count + first seen / last seen

**`/settings`**

- API URL (worker URL)
- Auth token
- Destination email address
- Toggle: auto-forward from allowed senders
- Toggle: auto-reject from blocked senders
- Connection status indicator

#### State Management

Use TanStack Query for server state:

- `useQuery(['emails', status])` — email list by status
- `useQuery(['senders'])` — sender list
- `useQuery(['settings'])` — settings
- `useMutation` for approve/reject/allow/block
- WebSocket messages invalidate relevant queries

Settings (API URL, token) stored in `chrome.storage.local`.

### 5. Real-Time Updates (Durable Object)

Single Durable Object instance acts as a WebSocket hub.

```
Email Worker                 Chrome Extension
    │                              │
    ├── POST /notify ──►  Durable Object
    │                        │
    │                        ├── broadcast to all
    │                        │   connected WebSockets
    │                        │          │
    │                        │          ▼
    │                        │   Extension receives:
    │                        │   { type: "new_email", email: {...} }
    │                        │   → invalidate pending query
```

## DNS & Email Setup

### For any domain (e.g., appulse.nl)

**Change:**

```
MX  appulse.nl  →  route1.mx.cloudflare.net  (priority 1)
MX  appulse.nl  →  route2.mx.cloudflare.net  (priority 2)
MX  appulse.nl  →  route3.mx.cloudflare.net  (priority 3)
```

**Keep (for outbound sending via Proton/any provider):**

```
TXT  appulse.nl  →  v=spf1 include:_spf.protonmail.ch ~all
CNAME protonmail._domainkey.appulse.nl → ...  (existing DKIM)
TXT  _dmarc.appulse.nl  →  v=DMARC1; p=none; ...
```

**Cloudflare Email Routing config:**

- Set up a catch-all route → Email Worker
- Or specific address routes (e.g., `contact@` → Email Worker)

## Key Decisions & Constraints

1. **Deferred forwarding requires SMTP relay.** Cloudflare's `message.forward()` only works inside the email event handler. For emails approved later, you need an external service to re-send. **Recommended: Resend** (simple API, free tier covers low volume, good deliverability).

2. **Auth is a shared bearer token.** This is a single-user tool — no need for OAuth or user accounts. Generate a random token, set it in the worker env, paste it into extension settings.

3. **No raw email storage in v1.** Body preview + headers are enough for the approval UI. Add R2 storage later if you want full email replay on forward.

4. **Classification is heuristic-first.** LLM classification adds latency and cost to every incoming email. Start with pattern matching, add AI as opt-in for unknowns.

5. **The extension is the only UI.** No web dashboard. This keeps it simple and always one click away.
